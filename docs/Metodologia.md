# 🧩 SCRUM + Pair Programming

## 📌 Estructura general

SCRUM (Organización del trabajo)

- **Sprint**
  - Historias de usuario
  - Desarrollo
    - 👉 Pair Programming (cómo programamos)

- **Daily Meeting**
  - Seguimiento de parejas y tareas

- **Sprint Review**
  - Presentación del trabajo realizado

- **Retrospective**
  - Evaluación del uso de Pair Programming

---

## 👨‍💻 ¿Qué es Pair Programming?

**2 PERSONAS → 1 ORDENADOR**

- **Driver** → escribe código  
- **Navigator** → revisa, piensa, corrige  

🔁 Cambio de roles cada 20-30 minutos

---

## 👥 Organización (equipo de 5)

✔ 2 parejas + 1 persona rotatoria  

| Día       | Pareja 1 | Pareja 2 | Rol extra     |
|----------|---------|---------|--------------|
| Lunes    | A + B   | C + D   | E (apoyo)    |
| Martes   | A + C   | B + E   | D (apoyo)    |
| Miércoles| A + D   | C + E   | B (apoyo)    |

🔁 Rotación continua → todos trabajan con todos

---

## ✅ Beneficios

- Menos errores  
- Mejor calidad de código  
- Aprendizaje constante  
- Código más uniforme  
- Menos bloqueos  
- Mejor comunicación  

---

## ⚠️ Obstáculos

- Diferencia de niveles  
- Cansancio mental  
- Conflictos de ideas  
- Sensación de lentitud  
- Mala organización  

---

## 🛠️ Claves para aplicarlo bien

- No usarlo en TODO → solo tareas importantes  
- Rotar parejas frecuentemente  
- Definir tiempos de cambio de rol  
- Integrarlo dentro del Sprint (SCRUM)  
- Evaluarlo en la retrospectiva  

---

## 💬Motto del proyecto

> “Se empleará SCRUM como marco de trabajo, incorporando Pair Programming como técnica de desarrollo para mejorar la calidad del código y fomentar el aprendizaje colaborativo.”
